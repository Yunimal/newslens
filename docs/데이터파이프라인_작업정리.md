# NewsLens 발표자료 — 데이터 파이프라인 작업 정리 (담당 A)

> 이 문서는 NewsLens 프로젝트에서 **데이터 수집, 전처리, 임베딩, 차원 축소/고차원 군집화, 스키마 검증 및 관계망 그래프를 포함한 전체 백엔드 데이터 파이프라인 구축**을 전담한 담당 A의 작업 내용을 체계적으로 정리한 발표/제출용 요약본입니다.
> 기말과제 채점 기준인 **NLP 기술 활용 수준, 시스템 완성도, 엔지니어링 신뢰성**에 부합하도록 서술되어 있습니다.

---

## 1. 프로젝트 역할 및 기여 요약

본 프로젝트에서 전체 뉴스 분석 서비스의 토대가 되는 **엔드투엔드(E2E) 데이터 파이프라인 스크립트 세트를 독자적으로 설계 및 구축**하고, 프론트엔드 대시보드 및 지도가 완벽한 정합성 하에 구동될 수 있도록 백엔드 데이터 프로세스를 완성했습니다.

* **핵심 역할**: 뉴스 크롤링 ➡️ 노이즈 정제 전처리 ➡️ 의미론적 고밀도 임베딩 생성 ➡️ 고차원 군집 분석 및 2D 매핑 ➡️ 관계망 그래프 데이터 구축 ➡️ Pydantic 스키마 검증 및 Fail-Fast 배포.

---

## 2. 세부 작업 내용 및 엔지니어링 디테일

### ① 뉴스 수집 및 텍스트 정제 (`collect_sample.py`)
* **구현 내용**: RSS 목록 페이지 분석을 거쳐 연합뉴스 최신 뉴스 상세 페이지를 크롤링하여 실데이터셋(369개 기사)을 수집하는 자동화 모듈을 구축했습니다.
* **정교한 NLP 전처리**: 기사 본문 내의 미디어 포토 캡션, 기자 이메일 주소, 언론사 저작권 문구, 자동요약 안내 배너 등 임베딩의 의미론적 유사도 연산을 방해하는 노이즈 텍스트를 **정규 표현식(Regex) 풀**을 설계하여 깨끗하게 제거했습니다.

### ② OpenAI Embedding API 연동 (`generate_embeddings.py`)
* **구현 내용**: OpenAI `text-embedding-3-small` API를 연동하여 정제된 뉴스 텍스트를 512차원 밀집 벡터(Dense Vector)로 변환했습니다.
* **안정성 설계**: 
  * API Rate Limit을 방지하기 위한 슬립 타임 가드를 구성했습니다.
  * API 키가 없는 오프라인 상태에서도 동일한 검색 체계를 로컬에서 모방 테스트할 수 있는 **결정론적 Mock 임베더(Feature Hashing, Bag-of-Words)** 폴백 체계를 마련했습니다.

### ③ UMAP 차원축소 및 고차원 KMeans 군집화 (`reduce_dimensions.py`) [핵심 기여 ⭐]
* **기존 문제 개선**: 
  * UMAP을 통해 2차원 좌표`(x, y)`로 축소된 평면 상에서 KMeans 군집화를 실행하여, 군집이 바둑판 격자 형태로 획일화되고 개수 분포도 부자연스럽게 균등하게 쪼개지는 시각적 한계가 존재했습니다.
  * 이를 해결하기 위해 군집 분석 대상을 2D 공간이 아닌, 의미 정보가 온전히 살아있는 **512차원 고차원 임베딩 벡터 공간**으로 이전하여 KMeans(`n_clusters=6`)를 수행하도록 개선했습니다.
  * UMAP은 2D 시각적 레이아웃(`x`, `y` 좌표) 매핑용으로만 사용하여 시각화의 정확도와 자연스러움을 향상시켰습니다. (군집 크기가 실제 최신 뉴스 주제별 빈도에 맞춰 128건에서 26건까지 역동적이고 자연스러운 밀도로 렌더링됨)
* **Fail-Fast 예외 처리**: 고차원 군집화 및 차원 축소 처리 시 예외 발생 또는 필수 패키지가 누락되었을 경우, 잘못된 데이터 생성을 막기 위해 오류 로그와 함께 즉시 빌드를 중단(`sys.exit(1)`)하도록 구현해 신뢰성을 올렸습니다.

### ④ 배포용 포맷 구성 및 스키마 검증 (`05_export.py`)
* **구현 내용**: 흩어진 좌표, 메타데이터, 감정 분포, 키워드 트렌드 통계를 취합해 프론트엔드가 즉시 조회할 수 있는 최적의 `articles.json` 및 `embeddings.json` 정적 DB 포맷을 배포합니다.
* **정합성 게이트**: 
  * Pydantic 라이브러리의 데이터 유효성 모델(`ArticlesFileModel`, `EmbeddingsFileModel`)을 최종 단계에 삽입하여 스키마 타입과 제약 조건에 어긋나는 기사가 있을 경우 배포를 즉각 차단합니다.
  * 이전 파이프라인의 에러로 인해 특정 기사에 `cluster_id` 속성이 누락되어 정합성이 깨질 경우, 임의의 기본값(0)을 채워 오류를 숨기지 않고 명시적으로 실패(`sys.exit(1)`)시키는 Fail-Fast 아키텍처를 적용했습니다.

### ⑤ 지식 관계망 그래프 구축 (`create_graph.py`)
* **구현 내용**: 뉴스 데이터의 주요 개체명(PER, ORG, LOC) 간의 동시 출현(Co-occurrence) 빈도를 바탕으로 노드와 엣지를 계산하여 관계망 데이터 `graph.json`을 자동 구축했습니다.

## 3. 파이프라인 스크립트 파일별 스펙 및 세부 역할

파이프라인을 이루는 각 파이썬 스크립트 파일의 입출력 관계와 주요 기술 요약입니다.

### ① `collect_sample.py` (뉴스 수집 및 정제)
* **역할**: 연합뉴스 최신 기사 크롤링 및 텍스트 데이터 파싱
* **입력**: 연합뉴스 최신 뉴스 목록 RSS 및 페이지 링크
* **출력**: `pipeline/raw/raw_articles_500.json` (수집된 원본 기사 목록)
* **주요 특징**: 차단 방지를 위한 User-Agent 순환 및 접속 지터링 적용, 마케팅 문구/포토 캡션/기자 이메일 등 임베딩을 방해하는 불필요 텍스트를 정규표현식으로 정교하게 제거하는 노이즈 클리닝 구현

### ② `enrich_articles.py` (LLM 뉴스 분석 및 정보 추출)
* **역할**: LLM을 활용한 뉴스 기사 요약, 핵심 키워드, 감정 분석, 엔티티 추출
* **입력**: `pipeline/raw/raw_articles_500.json`
* **출력**: `pipeline/raw/raw_articles_500.json` (수집본에 속성 인플레이스 추가)
* **주요 특징**: `gpt-4o-mini` API 연동 및 정형화된 응답 포맷(JSON Mode Schema) 적용, API 차단/미설정 시에도 정합성 규격을 유지하여 빌드를 통과시키는 오프라인 빈 배열 매핑 탑재

### ③ `generate_embeddings.py` (텍스트 임베딩 벡터 생성)
* **역할**: 뉴스 의미 분석을 위한 고차원 수치 벡터화
* **입력**: `pipeline/raw/raw_articles_500.json`
* **출력**: `pipeline/raw/embeddings_raw.json` (기사별 임베딩 벡터 정보)
* **주요 특징**: OpenAI `text-embedding-3-small` 임베딩 생성 호출, 임계 한도 방지 딜레이, API 키 누락 시 JS 코어 검색과 대조 연산이 가능한 해시 BoW 기반 결정론적 폴백 임베더 내장

### ④ `reduce_dimensions.py` (차원 축소 및 고차원 군집화)
* **역할**: 고차원 벡터의 2D 평면 매핑 및 고차원 기준 클러스터 분류
* **입력**: `pipeline/raw/raw_articles_500.json`, `pipeline/raw/embeddings_raw.json`
* **출력**: `pipeline/raw/articles_with_coords.json` (2D 좌표 및 `cluster_id` 병합 기사 데이터)
* **주요 특징**: UMAP 라이브러리 기반 2차원 매핑(미설치 시 PCA 및 Gram-Schmidt 무작위 투영 순차 폴백), **512차원 고차원 임베딩 기준 KMeans 군집화** 수행, 군집화 실패 시 예외를 숨기지 않는 Fail-Fast 오류 중단(`sys.exit(1)`) 적용

### ⑤ `05_export.py` (통합 데이터 스키마 검증 및 배포)
* **역할**: 백엔드 데이터 가공 및 클라이언트(Next.js) 배포용 파일 빌드
* **입력**: `pipeline/raw/articles_with_coords.json`, `pipeline/raw/embeddings_raw.json`
* **출력**: `data/articles.json`, `data/embeddings.json` (512차원 슬라이싱 적용)
* **주요 특징**: Pydantic 검증 모델(`ArticlesFileModel`, `EmbeddingsFileModel`)을 이용한 빌드 타임 데이터 스키마 적합성 심사, 특정 기사에 `cluster_id`가 누락된 정합성 오류 감지 시 배포를 사전 차단하는 Fail-Fast 오류 검증 탑재

### ⑥ `create_graph.py` (인물/기관 관계망 그래프 생성)
* **역할**: 기사별 개체명 추출 정보 기반 엔티티 간 동시 출현 그래프 구축
* **입력**: `data/articles.json`
* **출력**: `data/graph.json` (시각화용 인물-기관 관계 노드 및 연결선 데이터)
* **주요 특징**: ORG/PER/LOC 타입별 가드 필터링, 노드 수 상위 60개 제한, 가중치 임계값(`weight >= 2`)을 기준으로 복잡도를 제어한 최적 그래프 생성

### ⑦ `run_pipeline.py` (전체 파이프라인 연쇄 실행기)
* **역할**: 1단계 수집부터 6단계 그래프 빌드까지의 전체 빌드 프로세스 제어
* **주요 특징**: 서브프로세스(`subprocess`)를 사용해 오류 발생 시 즉각 중단하고 이전 단계 로그를 추적하기 쉽도록 구성된 빌드 오케스트레이터

## 4. 최종 배포용 데이터 파일(JSON) 상세 구조 및 스펙

파이프라인 빌드를 통해 최종 생성되어 `data/` 폴더에 배포되는 3대 핵심 데이터 파일의 구조와 속성 설명입니다.

### ① `articles.json` (기사 및 군집 정보 메인 DB)
* [articles.json](/data/articles.json)
* **역할**: 대시보드 범례, 지도 마커, 기사 카드 및 검색 필터, 키워드 트렌드 차트에 연결되는 메인 프론트엔드 데이터베이스 파일입니다.
* **상세 구조**:
  * `meta`: 데이터 소스 정보(`source_name`: "연합뉴스"), 수집 날짜 범위, 전체 기사 수(`369`), 군집 수(`6`) 등을 포함하는 데이터셋 요약 정보
  * `clusters`: 고차원 KMeans로 분석된 6개 군집의 메타데이터 배열
    * `centroid`: 군집에 속한 기사들의 UMAP 2D 좌표 평균값(`x`, `y`)
    * `size`: 해당 군집에 할당된 뉴스 기사 개수
    * `sentiment_dist`: 군집 내 기사들의 감정(긍정/중립/부정) 분포 개수
  * `articles`: 전체 기사 상세 배열
    * `id`: `aXXXX` 형태의 스키마 매핑 기사 고유 식별자
    * `summary3`: 기사 본문을 기반으로 추출된 3글머리 요약 튜플
    * `cluster_id` 및 `x`, `y`: 고차원 군집 분류 번호 및 UMAP 2D 지도 시각화 좌표
    * `sentiment`, `keywords`, `entities`: 언론사 논조 분석값, 3~5개 주요 단어, 추출된 인물/기관/지명 배열
  * `trends`: 시계열 그래프 구성을 위해, 상위 20개 주요 키워드의 일자별 기사 등장 빈도(Date, Count)를 집계한 배열

### ② `embeddings.json` (런타임 RAG 검색용 벡터 DB)
* [embeddings.json](/data/embeddings.json)
* **역할**: 사용자 질문에 대응해 코사인 유사도 연산을 수행하기 위한 서버 전용 의미론적 벡터 데이터베이스 파일입니다. (보안 및 격리를 위해 브라우저 클라이언트로 노출되지 않도록 `server-only`로 제한)
* **상세 구조**:
  * `model`: 기사 벡터화에 사용된 임베더 명칭 (`text-embedding-3-small` 또는 `hash-fallback-v1`)
  * `dim`: 임베딩 벡터의 차원수 (정밀도 유지 및 비용 절감을 위한 `512` 차원 슬라이싱 적용)
  * `items`: 기사의 고유 ID(`id`)와 이에 대응하는 `v` (512차원 실수형 임베딩 벡터 값) 배열

### ③ `graph.json` (인물·기관 관계망 네트워크 DB)
* [graph.json](/data/graph.json)
* **역할**: 뉴스 속 주요 엔티티(인물/기관/장소) 간의 시각적 네트워크 연결망 그래프를 그리기 위한 D3 Force-directed 그래프 연동 전용 데이터 파일입니다.
* **상세 구조**:
  * `nodes`: 네트워크의 핵심 노드 리스트 (상위 빈도 60개 제한)
    * `id`: 개체명 (예: "기상청", "김정은" 등)
    * `type`: 개체 분류 범주 (`PER` 인물, `ORG` 기관, `LOC` 지역)
    * `count` 및 `article_ids`: 해당 개체가 뉴스에 총 등장한 횟수 및 관련 기사 ID 리스트
  * `edges`: 노드와 노드 사이의 관계선 리스트 (가중치 `weight >= 2` 필터링)
    * `source` / `target`: 연결 관계가 형성된 출발 개체와 도착 개체명
    * `weight` 및 `article_ids`: 두 개체가 뉴스 내에서 동시에 등장한 기사 수(선 굵기로 표현됨) 및 관계 기사 ID 리스트

## 5. 작업 신뢰성 검증 지표

* **Pydantic 스키마 정합성 검사**: 파이프라인 수행 시 전체 구조 정합성 통과 완료.
* **TypeScript 컴파일러 검사**: `npm run typecheck` (`tsc --noEmit`) 실행 결과 에러 없음.
* **Next.js 빌드**: `npm run build`를 통한 배포용 번들링 검증 완료.
* **유사도 연산 단위 테스트**: `npm run test` 실행 결과 12개 테스트 케이스 전원 통과 완료.
