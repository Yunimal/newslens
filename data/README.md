# 가공된 데이터셋 저장소 (`data/`)

이 폴더는 Python 파이프라인 가공을 완료하고 프론트엔드 대시보드 및 AI 백엔드가 직접 로드하여 사용하는 **공식 배포용 데이터 파일**들이 적재되는 곳입니다.

---

## 📄 파일 및 데이터 명세

### 1. `articles.json` (클라이언트 공개용)
- **용도**: 지도 범례/툴팁, 드릴다운 패널, 기사 카드 및 필터, 키워드 트렌드 차트를 그리기 위한 메인 데이터 파일입니다.
- **주요 특징 및 정합성**:
  - `meta`: 수집 매체명, 총 기사 수 및 수집 기간을 나타냅니다.
  - `clusters[]`: KMeans 알고리즘으로 군집화된 6개 클러스터의 중심좌표(`centroid`), 소속 기사 수, 감성 분포(`sentiment_dist`) 등을 포함합니다.
  - `articles[]`: `"a" + 4자리` 기사 ID(`a0001`~`aXXXX`) 포맷, `published_at` (스네이크 케이스 규격 준수), `summary3` (정확히 3문장 배열) 등을 완벽히 만족합니다.
  - `trends[]`: 전체 기사 핵심 키워드 중 **빈도 상위 20개**에 대해 일별 등장 기사 수를 집계한 시계열 데이터(기사 수가 0인 날도 날짜 누락 없이 포함)가 포함되어 있습니다.

### 2. `coords.json` (클라이언트 공개용)
- **용도**: 프론트엔드 지도 렌더링 최적화를 위해 좌표 데이터만 따로 분리한 초경량화 JSON 파일입니다.
- **구조**: `[ { "articleId": "aXXXX", "x": float, "y": float }, ... ]`
- **특징**: 기존에 `embeddings.json` 파일에 좌표를 오버라이팅하여 발생하던 백엔드 간 파일명 충돌을 완벽히 방어하고, 클라이언트 단의 빠른 데이터 수신을 돕습니다.

### 3. `embeddings.json` (⚠️ 서버 전용 - 클라이언트 전송 금지)
- **용도**: `POST /api/ask` 질문답변(RAG) 서비스 작동 시 백엔드 검색기(Retriever)가 임베딩 유사도를 계산하기 위해 사용하는 밀도 벡터 아카이브입니다.
- **주요 규격**:
  - `model`: 임베딩에 사용된 모델명. 파이프라인 구동 시 `.env`에 `OPENAI_API_KEY`가 지정되었는지 여부에 따라 `"hash-fallback-v1"` 혹은 `"text-embedding-3-small"`로 유동적/자동 반영됩니다.
  - `dim`: **`512`** (RAG 검색 비용 절감 및 규격을 위한 **512차원 제한 고정**)
  - `items[].v`: 1536차원에서 512차원으로 슬라이싱되고, **소수점 4자리로 반올림** 적용된 float 벡터 데이터입니다.
